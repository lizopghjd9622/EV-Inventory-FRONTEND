# 智能销售库存系统 (Smart Sales Inventory System) - 前端 Spec

## 1. 简介

本系统是面向门店/仓库操作人员的极简 **智能销售库存 Web 应用**。核心设计理念是 **"语音驱动、最短路径"**：用户登录后只需长按按钮说话，系统自动识别商品并生成单据，用户核对确认即完成操作。整体界面保持极简风格，减少认知负担。

---

## 2. 用户故事 (User Stories)

**故事 1：登录**
> **作为** 操作人员
> **当** 我打开应用时
> **我希望** 输入手机号和密码后保持登录状态（关闭浏览器后再次打开无需重新登录）
> **以便** 每天快速进入操作界面，避免重复认证

**故事 2：语音录入销售单**
> **作为** 销售人员
> **当** 我在主页长按「销售」按钮说出商品信息时
> **我希望** 系统自动识别我说的商品和数量，逐条展示在列表中，结束后让我核对确认
> **以便** 无需手动逐条录入，提升录单效率

**故事 3：语音录入进货单**
> **作为** 仓库/采购人员
> **当** 我在主页长按「进货」按钮说出采购内容时
> **我希望** 系统自动识别商品并生成进货单供我核对
> **以便** 快速完成进货入库操作

## 3. 页面与交互规格

### 3.1 登录页 (Login)

**交互说明：**
- 表单包含两个字段：**手机号**、**密码**。
- 点击「登录」按钮发起认证请求。
- **验证码**：当密码连续输入错误时，展示图形验证码，用户需填写验证码后方可继续尝试。
- **登录态持久化**：登录成功后将 Token 持久化到 `localStorage`，关闭浏览器重新打开时自动恢复登录状态，无需重新输入账号密码。
- 登录成功后跳转到**操作主页**。

**错误处理：**

| 场景 | 展示 |
| :--- | :--- |
| 手机号格式错误 | 字段下方红色提示文案 |
| 密码错误（前几次） | Toast 提示"手机号或密码错误" |
| 密码错误（触发阈值后） | 表单下方展示图形验证码输入框 |
| 验证码错误 | 提示"验证码错误"并刷新验证码 |

---

### 3.2 操作主页 (Home)

**布局：**
- 登录后直接进入此页，全屏居中展示两个大按钮：
  - **「销售」**
  - **「进货」**
- 无多余导航、菜单或其他功能入口，保持极简。

---

### 3.3 语音录入流程（销售 & 进货通用）

销售和进货的前端交互逻辑完全一致，仅调用的接口和单据类型不同。

#### 步骤 1：长按录音

- 用户**长按**「销售」或「进货」按钮开始录音，**松开**立即停止录音。
- 录音过程中展示录音状态反馈（如动态波形动画或按钮状态变化）。
- **最大录音时长为 60 秒**，到达上限时自动停止录音并立即触发发送，同时提示用户"已达最大时长，自动发送"。

#### 步骤 2：上传音频 & 流式识别

- 录音结束后，前端将完整音频文件**一次性上传**至后端流式接口（`POST /sales/stream` 或 `POST /purchase/stream`）。
- 请求发出后，页面进入**流式识别状态**：
  - 展示加载动画（如旋转图标或骨架屏条目）表示正在处理。
  - 后端每识别出一个商品，以 **SSE / Chunked 流** 返回单条数据。
  - 前端实时将每条商品**逐条追加**到列表底部，形成流式渲染效果。
  - 列表每次追加时带有短暂的入场动画（如淡入或下滑）。

#### 步骤 3：流式完成后 → 自动进入详情页

- 后端流式响应结束后，页面**自动跳转**到对应的单据详情页。
- 详情页展示所有已识别的商品条目列表。

#### 步骤 4：详情页核对与编辑

- 用户可在详情页对每条商品进行编辑：
  - **商品名称**（可修改）
  - **数量**（可修改）
  - **单位**（可修改）
- 支持**删除**单条商品条目。

#### 步骤 5：操作选择

详情页底部提供两个操作按钮：

| 按钮 | 行为 |
| :--- | :--- |
| **重新录音** | 清空当前识别结果，返回主页重新触发语音录入 |
| **确认提交** | 调用确认接口（`POST /sales/:id/confirm` 或 `POST /purchase/:id/confirm`），成功后**返回操作主页** |

#### 错误处理

| 场景 | 展示与操作 |
| :--- | :--- |
| 流式接口请求失败 / 中途断开 | 展示错误提示，提供「重新发送」按钮，点击后重新上传**上一次的音频**并重试 |
| 确认接口失败 | Toast 提示错误信息，留在详情页，用户可再次点击确认 |
| 网络超时 | 提示"网络异常，请检查连接后重试"，提供重试按钮 |

## 4. 职责边界 (Scope)

### 4.1 系统包含 (In Scope)
- **登录**：手机号+密码，Token持久化，验证码保护。
- **操作主页**：仅含销售、进货两个入口按钮。
- **语音录入**：长按录音（最长60秒），完整音频上传。
- **流式展示**：逐条追加商品列表，加载动画。
- **单据详情**：商品信息可编辑（名称、数量、单位），支持删除条目。
- **重新录音**：允许放弃当前结果重新开始。
- **确认提交**：调用确认接口，成功返回主页。
- **错误重试**：流式失败时可重发上一次音频。

### 4.2 系统不包含 (Out of Scope)
- 注册、忘记密码等用户管理界面（账号由后台创建）。
- 历史单据查询与列表页。
- 库存查询页面。
- 多语言支持。
- 离线能力。

## 5. 非功能性需求

| 编号 | 描述 |
| :--- | :--- |
| N-01 | 页面风格极简，主色调清晰，主操作按钮视觉权重最高 |
| N-02 | 流式渲染过程中 UI 不得卡顿，逐条追加需流畅 |
| N-03 | Token 存储于 `localStorage`，页面加载时自动校验有效性，失效则跳回登录页 |
| N-04 | 录音按钮的长按体验需兼容移动端触摸事件（`touchstart` / `touchend`）和 PC 鼠标事件 |
| N-05 | 音频上传期间展示上传进度或 loading 状态，防止用户重复触发 |

## 7. 验收标准 (Acceptance Criteria)

### Case 1：登录态持久化
- [ ] 用户登录成功后，关闭浏览器标签页。
- [ ] 重新打开应用，**直接进入操作主页**，无需重新输入账号密码。

### Case 2：验证码触发
- [ ] 连续输入错误密码达到阈值后，页面展示图形验证码输入框。
- [ ] 正确填写验证码后方可继续尝试登录。

### Case 3：长按录音 & 超时自动发送
- [ ] 长按「销售」按钮，录音开始，展示录音状态动画。
- [ ] 松开按钮，录音停止，音频开始上传。
- [ ] 继续长按超过 60 秒，录音自动停止并上传，页面提示"已达最大时长，自动发送"。

### Case 4：流式逐条追加
- [ ] 音频上传后，页面展示加载动画。
- [ ] 后端每返回一个商品条目，列表中即时出现该条目（带入场动画）。
- [ ] 流式结束后，自动跳转到详情页。

### Case 5：详情页编辑
- [ ] 在详情页修改某商品的名称、数量、单位后，数据正确保存到提交内容中。
- [ ] 删除某条商品后，该条目从列表中移除，确认提交时不包含已删除条目。

### Case 6：重新录音
- [ ] 在详情页点击「重新录音」，清空当前列表，返回操作主页。
- [ ] 再次长按按钮，可正常录音并重新识别。

### Case 7：流式错误重试
- [ ] 流式接口请求失败，页面展示错误提示及「重新发送」按钮。
- [ ] 点击「重新发送」，自动使用上一次的音频重新请求，无需再次录音。

### Case 8：确认成功返回主页
- [ ] 在详情页点击「确认提交」，接口调用成功。
- [ ] 页面自动返回**操作主页**，两个按钮正常显示，可继续下一次操作。
